# SQL Portfolio: Database Design, Normalization & Advanced Querying

This repository contains the **Final Integrated Project (TFI)** developed as part of the Database Diploma at **Ícaro**. The project demonstrates practical skills in the complete lifecycle of relational database design, data normalization, and writing optimized queries for business decision-making.

## 🛠️ Tech Stack & Environment
* **Database Engine:** MySQL / MariaDB
* **Modeling & Client Tools:** DBeaver / MySQL Workbench
* **Operating System:** Ubuntu Linux

---

## 📂 Project Structure

### 1. 📊 Relational Database Design & Normalization (`/modelo_normalizado_business`)
Hands-on implementation transforming raw, unstructured tabular data into an optimized relational model by strictly applying **First, Second, and Third Normal Forms (1NF, 2NF, 3NF)**.
* **Artifacts:** Structure definition SQL scripts (`.sql`), Entity-Relationship Models (ERM), and Physical Diagrams (ERD) exported from MySQL Workbench.
* **Use Case:** A relational schema named `business` tailored for handling customer orders, managing products, and maintaining strict referential integrity.

### 2. 🔍 Advanced SQL Queries & Business Logic (`/consultas_avanzadas_jardineria`)
Implementation of a comprehensive business database model (`jardineria`) designed to extract analytical insights from commercial operations.
* **Artifacts:** Database setup scripts, schema diagrams mapped via DBeaver, and an extensive analytical query file (`Consultas-DavidAlmaraz.sql`).
* **Demonstrated Skills:** Data filtering (`WHERE`), conditional logic, complex multi-table relationships (`JOIN` operations), aggregations, grouping functions, and query optimization.

### 3. 🧠 Conceptual Foundations (`/fundamentos_teoricos`)
Technical documentation outlining database architectural decisions. It explains key concepts such as the distinction between conceptual (ERM) and logical/physical (ERD) modeling, the strategic use of Primary and Foreign Keys, and data normalization criteria.
